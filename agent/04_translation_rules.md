# 翻译规则

本文件只保存翻译风格、输出格式和模型行为约束，不保存长剧情和术语细节。已确认专有名词与 ASR 易错词以 `agent/03_glossary_confirmed.md` 为准，待确认风险词以 `agent/04_glossary_uncertain.md` 为准。

## 总原则

- 目标语言为自然、流畅、简洁的简体中文。
- 保留原对白意思，不添加解释性旁白。
- 不改变字幕时间轴。
- 不输出模型解释、注释或分析文字。
- 同一人物、地点、机构和称谓在同一集中必须保持一致。
- 粤语语气词可适度简化，不机械直译。
- 粗口、讽刺、威胁、亲昵称呼应按剧情语气自然处理。
- 旧期香港语境中的称呼要符合时代感，不要翻成现代网络表达。

## 粤语到简体中文风格

- 口语自然，但不过度现代化。
- 商战、家族、职场、社交场景用语要稳重。
- 情绪激烈的对白可以保留力度，但避免夸张。
- 称谓优先保留人物关系，例如老板、少爷、太太、经理、先生、小姐。
- 对白中重复的语气词可压缩，但不要删掉关键情绪。
- 粤语口语在普通话中没有直接对应表达时，应意译。
- 译文要适合字幕阅读，不要过长、绕口或书面腔过重。

## 常见粤语词处理

- “唔”按语境译为“不、没、不会、不是”。
- “冇”按语境译为“没有、没”。
- “咁”按语境译为“这么、那样、那就”。
- “佢”译为“他、她、这人”，按上下文判断。
- “你哋 / 我哋 / 佢哋”译为“你们 / 我们 / 他们”。
- “得唔得”译为“行不行、可以吗、能不能”，按语气选择。
- 已确认行业词、饮食词和高风险 ASR 易错词，按 `agent/03_glossary_confirmed.md` 处理，不在本文件重复解释。

## DeepSeek 翻译流程约束

- `pre_correction_context` 只服务 ASR 文本校正，不输出最终字幕译文。
- `deepseek_context` 只服务翻译上下文摘要，不修改 ASR 文本。
- 逐条翻译时必须参考当前批次 `deepseek_context` 摘要。
- 逐条翻译时默认参考当前行前 3 条、后 4 条原文；窗口大小由 `deepseek.translation_window_before` 和 `deepseek.translation_window_after` 控制。
- 逐条翻译时同时参考术语表、风格规则和角色名资料。
- 输出只给单条简体中文译文，不夹带说明、分析或注释。

`deepseek_context` 摘要应提取剧情进展、人物关系、称谓、语气和可能的专有名词；逐条译文必须含义准确、语气自然、长度适合字幕，并可被程序解析。

## 输出格式要求

双语 SRT：

```text
粤语原文
简体中文译文
```

纯中文 SRT：

```text
简体中文译文
```

所有最终字幕应通过质量检查：

- 序号连续。
- 时间轴不倒退。
- 无空译文。
- 双语版和中文版条数一致。
- 专有名词尽量一致。
